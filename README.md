# レッスンレポート自動化ツール — 進捗まとめ

最終更新: 2026年4月7日

---

## 1. プロジェクト概要

### 目的
ドラムレッスンの内容を Notion にまとめたデータをもとに、**生徒ごとのレッスンレポートを自動生成・配布する**ツールを作る。

### 職業・背景
- 職業: ドラム講師
- レッスン管理ツール: Notion

### Notion に記録している項目
| 項目 | 内容 |
|---|---|
| 生徒名 | レッスンを受けた生徒の名前 |
| 日付 | レッスン実施日 |
| 課題曲 | 現在取り組んでいる曲 |
| 本日のレッスン内容 | その日に行った練習・指導内容 |
| 今日のよかったところ | 生徒の良かった点 |
| もっとよくできる点 | 改善点・次の課題 |
| 次回のレッスン内容 | 次回の予定・取り組み事項 |
| 次回レッスン日 | 次回のレッスン予定日 |

---

## 2. デザイン要件

### スタイル
- 温かみのあるシンプルデザイン
- どこかスタイリッシュ
- **深緑（forest green）をブランドカラー**として使用

### 対象生徒の年齢層
- 子ども（小学生以下）
- 中高生
- 大人（20〜40代）

### フォーマット
- **縦長（390px幅）** — スマホ向け、LINE・Instagram ストーリーに近いサイズ感

### 配布方法
- まずは**メールで自分（講師）に届く**形から始める
- 将来的に生徒へ画像で配布

### ブランド要素
- 特になし（シンプルでOK）

---

## 3. デザイン決定事項

### カラーパレット
| 役割 | カラー |
|---|---|
| ヘッダーグラデーション | `#0c2411` → `#1e5c28` → `#2d7a38` |
| カード背景 | `#faf8f4`（温かいオフホワイト）|
| ページ背景 | `#e8e2da`（ベージュ）|
| Good セクション | Amber（`bg-amber-400`）|
| Next Step セクション | Sky Blue（`bg-sky-400`）|
| Next Lesson セクション | 薄緑（`bg-forest-50`）|

### フォント
- 見出し: **Playfair Display**（セリフ体、イタリック使いでスタイリッシュに）
- 本文: **Noto Sans JP**（日本語対応、軽量で読みやすい）

### アイコン
- 絵文字は**文字化けリスクがあるためすべて廃止**
- インラインSVGアイコンに統一（フォント・環境非依存）

### フッター
- 「お疲れさまでした」ピル（深緑バッジ）
- 「また次回も一緒に頑張りましょう。」サブテキスト
- 回数バッジ（#24 など）は不要 → 削除
- 講師からのひとことセクションは不要 → 削除

## 5. レビュー結果（サブエージェント）

### UI/UX デザイナー視点

#### 良い点
- ヘッダーで「誰の・何のレポートか」が即座に伝わる
- 深緑×ベージュで温かみと落ち着きが一貫している
- Good / Next の2カラム対比が情報として有効
- フッターが承認・励ましのトーンで好印象

#### 要改善点
| 課題 | 提案 |
|---|---|
| ラベルが英語のみで読み取りに一瞬かかる | 日本語主 + 英語小の表記に変更（例: 「本日のレッスン *Today's Lesson*」） |
| 補助テキストのコントラストが薄い | `text-white/55` → `text-white/75` に引き上げ |
| ラベル文字が小さすぎる | `text-[9px]` / `text-[10px]` → 最小 `text-[11px]` に統一 |
| 本文が細い環境がある | `font-light` → `font-normal` を検討 |
| 横パディングが混在（px-5/6/7） | `px-6` に統一してリズムを揃える |

---

### フロントエンドエンジニア視点

#### 良い点
- 幅 390px 固定で PNG 化と相性がよい
- `tailwind.config` でカラー・フォントを集約しており差し替えしやすい
- テンプレートエンジンでそのまま変数化できる構造

#### 要改善点
| 課題 | 対策 |
|---|---|
| Tailwind CDN は本番・CI に不向き | CLI ビルドで静的 CSS に切り替え、HTML と同梱する |
| Google Fonts の読み込みタイミングがブレる | Playwright で `waitForLoadState('networkidle')` を使う、またはフォントをローカルに |
| Notionデータをそのまま埋めるとXSSリスク | テンプレートエンジンの自動エスケープ or DOMPurify でサニタイズ |
| `div` だらけで意味がない | `article` / `header` / `section` / `time datetime=""` に置き換え |
| 装飾SVGにラベルがない | `aria-hidden="true"` を付けてアクセシビリティ向上 |

## 6. システム構成（4パーツ）

調査・検討の結果、以下の構成をベストプラクティスとして採用する。

```
┌──────────────────────────────────────────────────────┐
│                   GitHub Actions                      │
│                                                       │
│  [パーツ1] トリガー                                    │
│   cron スケジュール（毎レッスン後の時間帯）              │
│   または workflow_dispatch（手動実行）                  │
│              │                                        │
│              ↓                                        │
│  [パーツ2] ソース：Notion API                          │
│   @notionhq/client で当日レコードを取得                 │
│              │                                        │
│              ↓                                        │
│  [パーツ3] 処理：Playwright で HTML → PNG              │
│   HTMLテンプレートにデータ注入                          │
│   Playwright でスクリーンショット（390px縦長）           │
│              │                                        │
└──────────────┼───────────────────────────────────────┘
               │
               ↓
      [パーツ4] LINE Messaging API
      自分の LINE に PNG 画像をプッシュ送信
```

### パーツ1：トリガー — GitHub Actions cron

| 項目 | 内容 |
|---|---|
| 方式 | `schedule` イベント（cron 式）+ `workflow_dispatch`（手動） |
| 無料枠 | プライベートリポジトリ: **2,000分/月**、パブリック: **無制限** |
| 1回の実行時間 | 約2〜3分 → 月100回実行しても300分、**無料枠内で問題なし** |

```yaml
on:
  schedule:
    - cron: '0 11 * * *'  # 毎日20時JST（UTC+9）
  workflow_dispatch:
```

### パーツ2：ソース — Notion API

| 項目 | 内容 |
|---|---|
| SDK | `@notionhq/client` v5.x（公式 Node.js SDK） |
| 注意 | 2025年9月以降、`databases.query()` は非推奨。**`dataSources.query()` を使う** |
| 認証情報 | GitHub Secrets に `NOTION_TOKEN` と `NOTION_DATABASE_ID` を登録 |

### パーツ3：処理 — Playwright

| 項目 | 内容 |
|---|---|
| ライブラリ | `playwright`（Chromium のみインストールで十分） |
| テンプレート | HTML テンプレートに文字列置換でデータ注入（Handlebars も可） |
| 本番対応 | Tailwind CDN → **CLI ビルドの静的 CSS に切り替える**（CI の安定性のため） |
| **必須対応** | `fonts-noto-cjk` を apt でインストールしないと**日本語が文字化けする** |

```yaml
- name: 日本語フォントをインストール
  run: |
    sudo apt-get install -y fonts-noto-cjk
    sudo fc-cache -fv
```

### パーツ4：届ける先 — LINE Messaging API

| 項目 | 内容 |
|---|---|
| エンドポイント | `POST https://api.line.me/v2/bot/message/push` |
| 画像送信の注意 | **公開 HTTPS URL が必要**（ローカルファイルは直接送れない） |
| 解決策（最もシンプル） | パブリックリポジトリに PNG をコミット → `raw.githubusercontent.com` の URL を使用 |
| 認証情報 | GitHub Secrets に `LINE_CHANNEL_ACCESS_TOKEN` と `LINE_USER_ID` を登録 |

### 必要な GitHub Secrets 一覧

| キー名 | 内容 |
|---|---|
| `NOTION_TOKEN` | Notion Integration のシークレットトークン |
| `NOTION_DATABASE_ID` | レッスン記録データベースの ID |
| `LINE_CHANNEL_ACCESS_TOKEN` | LINE 公式アカウントのチャンネルアクセストークン |
| `LINE_USER_ID` | 自分の LINE ユーザー ID（`U` から始まる文字列）|

```
  GitHub リポジトリ作成 & 環境準備
  ├─ lesson-report/ をリポジトリとして初期化
  ├─ GitHub Secrets に4つのキーを登録
  ├─ Tailwind CSS CLI ビルド環境を整える
  └─ HTMLテンプレートの変数プレースホルダーを定義
       例: {{studentName}} {{date}} {{lessonContent}} ...

Phase 4
  Notion API 連携スクリプト作成
  ├─ @notionhq/client で当日レコードを取得
  ├─ dataSources.query() + 日付フィルターで絞り込み
  └─ 取得データをHTMLテンプレートに注入して report.html を生成

Phase 5
  Playwright で HTML → PNG 変換
  ├─ fonts-noto-cjk インストール（文字化け対策）
  ├─ page.setContent() + page.screenshot() で report.png を生成
  └─ 生成した PNG をリポジトリにコミット（LINE送信用の公開URL確保）

Phase 6
  GitHub Actions ワークフロー構築 & LINE 送信
  ├─ .github/workflows/generate-report.yml 作成
  ├─ cron スケジュール設定（毎レッスン後の時間帯）
  ├─ LINE Messaging API で自分宛てに PNG をプッシュ送信
  └─ 動作確認・エラーハンドリング追加

Phase 7（将来）
  生徒への直接配布
  └─ 生徒ごとに LINE_USER_ID を管理し、個別送信に拡張
```
